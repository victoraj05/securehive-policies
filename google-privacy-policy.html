1. Introduction

SecureNest ("we," "our," or "us") is committed to protecting your privacy. This Privacy Policy explains how we collect, use, disclose, and safeguard your information when you use our mobile application SecureNest ("the App").

2. Information We Collect

2.1 Account Information

  • Email Address: Used for account creation and recovery
  • User ID: Unique identifier for your account
  • Account Creation Date: For account management purposes

2.2 Encrypted User Data

  • Passwords: Stored in encrypted format using AES-256-CBC encryption with unique initialization vectors
  • Notes: Encrypted using the same AES-256 standard and stored securely
  • Categories: User-created categories for organization (encrypted)
  • URLs: Associated website URLs (encrypted)
  • Security Questions: Encrypted security questions and answers

2.3 Technical Information

  • Device Information: Device type, operating system version
  • App Usage Data: Features used, crash reports, performance metrics
  • IP Address: For security and fraud prevention
  • Session Data: Login times and app usage patterns

2.4 Biometric Data

  • Fingerprint/Face ID: Stored locally on device only (never transmitted)
  • Biometric Authentication Preferences: Whether biometric auth is enabled

3. How We Use Your Information

3.1 Primary Uses

  • Service Delivery: Providing password management functionality
  • Account Management: Creating and maintaining your account
  • Security: Protecting against unauthorized access and fraud
  • Sync: Synchronizing data across your devices

3.2 Secondary Uses

  • Analytics: Understanding app usage to improve features
  • Customer Support: Responding to your inquiries and issues
  • Legal Compliance: Meeting legal and regulatory requirements
  • Service Improvement: Developing new features and improvements

4. Zero-Knowledge Architecture

4.1 Encryption Model

  • All sensitive data is encrypted locally on your device using AES-256-CBC with random initialization vectors before transmission
  • We use your master password with SHA-256 hashing to derive 256-bit encryption keys
  • Each encryption operation uses a unique 16-byte initialization vector (IV) for enhanced security
  • We cannot access your decrypted passwords or notes due to zero-knowledge architecture
  • Even our employees cannot view your sensitive information
  • All network communications are secured with TLS 1.3 encryption

4.2 Master Password

  • Your master password is never transmitted to our servers
  • We cannot recover your master password if lost
  • Only you can decrypt your data with your master password

5. Data Storage and Security

5.1 Storage Locations

  • Cloud Storage: Encrypted data stored on secure cloud servers (AWS/Google Cloud)
  • Local Storage: Some data cached locally on your device
  • Geographic Location: Data may be stored in multiple regions for redundancy

5.2 Security Measures

AES-256 TLS 1.3 Zero-Knowledge Regular Audits
  • Encryption: AES-256-CBC encryption with 256-bit keys derived from SHA-256 hashing for all sensitive data
  • Transport Security: TLS 1.3 with automatic fallback to TLS 1.2 for secure data transmission
  • Key Management: Secure key storage using iOS Keychain and Android Keystore
  • Biometric Protection: Device biometric authentication (fingerprint, Face ID) for app access
  • Access Controls: Strict access controls for our servers
  • Regular Audits: Security audits and penetration testing
  • Backup Security: Encrypted backups with geographic distribution
